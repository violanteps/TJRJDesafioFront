Resumo do Projeto TJRJ - Desafio Técnico

Tecnologias Utilizadas

  • LocalDB: Utilizado como banco de dados local para armazenar as informações da aplicação.
  • Angular 18: Framework utilizado para o desenvolvimento do frontend.
  • .NET 8: Framework usado para o desenvolvimento do backend, oferecendo suporte para a criação de APIs RESTful e outras funcionalidades essenciais.
  • Bootstrap: Biblioteca de estilos CSS utilizada para criar um layout responsivo.
  • JsPDF: Biblioteca JavaScript usada para gerar relatórios diretamente no frontend.
  • Swagger: Usado para documentar e testar os endpoints das APIs.
  • AutoMapper: Usado no mapeamento de objetos entre diferentes camadas da aplicação.
  • ILogger: Implementado para registrar logs em diversas camadas da aplicação, ajudando a rastreabilidade e identificação de problemas.

Padrões de Projeto Adotados

  • Componente Service no Front para chamadas de endpoints: Criei um serviço Angular centralizado que é responsável por todas as chamadas aos endpoints da API, promovendo a reutilização de código e a separação de responsabilidades.
  • Componente Header no Front: Implementado para gerenciar a navegação e exibir informações constantes em todas as páginas.
  • Componente Footer no Front: Criado para exibir informações de rodapé e manter a consistência visual em todas as páginas.
  • History state: Utilizado para passar dados entre diferentes componentes no Angular sem expor os parâmetros na URL.

Arquitetura em Camadas no Backend

  • Controller: Camada responsável por receber as requisições HTTP e enviar as respostas, delegando a lógica de negócios para o serviço.
  • Service: Contém a lógica de negócios da aplicação e coordena as chamadas ao repositório.
  • Repository: Realiza o acesso aos dados através de operações CRUD diretamente no banco de dados.
  • Dapper: Ferramenta de micro-ORM utilizada na camada de persistência para executar queries e mapeá-las para objetos.
  • Controle de Transações: Implementado para garantir a consistência dos dados, especialmente em operações que envolvem múltiplas etapas.
  • Tratamento de erros: Implementado para garantir a detecção e rápida solução de possíveis erros no acesso aos dados.

Funcionalidades

  • CRUD Completo: Implementação das operações de criar, ler, atualizar e deletar para as entidades Livro, Autor, Assunto e Valor Livro.
  • Relatórios: Geração de relatórios agrupados por autor, exportáveis através de funcionalidade em tela.
  • Interface Responsiva: Desenvolvimento de uma interface de usuário moderna e adaptável a diferentes dispositivos, utilizando Bootstrap.

Considerações Adicionais

No modelo de dados inicial, foram adicionados os campos statusReg, DataCriacao e UltimaAtualizacao, com o objetivo de evitar exclusões físicas dos dados e proporcionar uma melhor rastreabilidade, além de facilitar a análise de auditoria de dados. Em um modelo mais robusto, poderia ter sido incluído o log de usuário e uma tabela de log de alterações para rastrear os dados que foram modificados, oferecendo um controle ainda maior sobre as operações realizadas.
